# CDF workflow deploy artifacts

Generated by **`python module.py transform build`** (or the operator UI **Build** action) from canvas definitions under **`transform/workflow_definitions/`**.

Default (unscoped) build writes flat files under this folder:

| File | Cognite resource |
|------|------------------|
| `etl_{id}.Workflow.yaml` | Workflow |
| `etl_{id}.WorkflowVersion.yaml` | WorkflowVersion |
| `etl_{id}.WorkflowTrigger.yaml` | WorkflowTrigger |
| `etl_{id}.config.yaml` | Trimmed scope document (local run + trigger `input`) |

With **`--scoped`**, artifacts go under **`workflows/<scope>/`** using `etl_{id}.{scope}.*` names.

Authoring sources stay under **`transform/workflow_definitions/`**. Toolkit **DataSet**: **`data_sets/ds_discovery_etl.DataSet.yaml`**. See **`transform/docs/BUILD.md`**.
